# DevOps Project 77

В рамках проекта реализовано развёртывание веб-приложения с использованием подхода **Infrastructure as Code**.

Инфраструктура полностью автоматизирована и разворачивается с нуля с помощью Terraform и Ansible.

---

## Инфраструктура

В проекте настроено:

* два веб-сервера с приложением
* балансировщик нагрузки с HTTPS
* база данных PostgreSQL
* домен с DNS-настройкой
* мониторинг и алерты

Приложение доступно по адресу:

https://mannkadevops.ru

---

## Используемые технологии

* Terraform — создание инфраструктуры
* Ansible — настройка серверов и деплой
* Docker — запуск приложения
* Datadog — мониторинг и алерты
* Upmon — проверка доступности

---

## Процесс развёртывания

1. Terraform создаёт инфраструктуру в облаке
2. Ansible настраивает сервера
3. Приложение запускается в Docker-контейнере
4. Настраивается мониторинг и проверка доступности

---

## Основные команды

```bash id="cmds77"
make terraform-setup
make terraform-plan
make terraform-apply
make terraform-destroy

make ansible-setup
make prepare
make deploy
make monitoring
```

---

## Мониторинг

На серверах установлен Datadog Agent.
Настроен монитор, проверяющий доступность приложения по HTTP.

Дополнительно используется внешний мониторинг через Upmon.

---

## Домен

Домен настроен и указывает на балансировщик нагрузки.
Подключён SSL-сертификат, обеспечивающий доступ по HTTPS.

---

## Безопасность

Секретные данные (ключи, токены, пароли) не хранятся в репозитории и передаются через переменные и Ansible Vault.

---

## Итог

Проект демонстрирует полный цикл:

* создание инфраструктуры
* автоматическую настройку серверов
* деплой приложения
* настройку мониторинга и доступности

Вся инфраструктура управляется кодом и может быть воспроизведена в любой момент.
